# 📖 Manual de Usuario - QA Test Generator

*Guía completa para usar el Generador de Casos de Prueba QA*

---

## 🎯 Introducción

El **QA Test Generator** es una herramienta de línea de comandos que utiliza inteligencia artificial para generar automáticamente historias de usuario y casos de prueba en formato SCRUM, soportando tanto inglés como español.

### ⏱️ Tiempo de Lectura: 15 minutos
### 🎯 Nivel: Usuario Intermedio
### 🛠️ Requisitos: Python 3.9+, API keys de IA

## 🚀 Inicio Rápido

### En 5 Pasos Estás Listo

```bash
# 1. Instalar dependencias
pip install -r requirements.txt

# 2. Configurar API keys
cp .env.example .env
# Editar .env con tus claves de API

# 3. Preparar archivos de entrada
# Crear data/user_story.txt con tu historia de usuario
# Verificar data/prompt_examples.json

# 4. Ejecutar
python -m src.main

# 5. ¡Listo! Sigue las instrucciones en pantalla
```

**Resultado esperado**: Archivos generados en `output/` con historias de usuario y casos de prueba en EN/ES.

## ⚙️ Instalación y Configuración

### Paso 1: Requisitos del Sistema

| Componente | Versión Mínima | Recomendado |
|------------|----------------|-------------|
| **Python** | 3.9.0 | 3.11+ |
| **RAM** | 4GB | 8GB+ |
| **Almacenamiento** | 500MB | 1GB+ |
| **Internet** | Estable | Alta velocidad |

### Paso 2: Instalación

#### Opción A: Usando pip (Recomendado)
```bash
# Clonar repositorio
git clone <url-del-repositorio>
cd qa-test-generator

# Instalar dependencias
pip install -r requirements.txt

# Verificar instalación
python -c "import src.main; print('✅ Instalación exitosa')"
```

#### Opción B: Usando Poetry
```bash
# Instalar Poetry si no lo tienes
curl -sSL https://install.python-poetry.org | python3 -

# Instalar dependencias
poetry install

# Activar entorno virtual
poetry shell
```

### Paso 3: Configuración de API Keys

#### Obtener API Keys

**Para Google Gemini:**
1. Ve a [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Crea una nueva API key
3. Copia la key

**Para OpenAI:**
1. Ve a [OpenAI Platform](https://platform.openai.com/api-keys)
2. Crea una nueva API key
3. Copia la key

#### Configurar Variables de Entorno

```bash
# Crear archivo .env
cp .env.example .env

# Editar .env con tus claves
GEMINI_API_KEY=AIzaSyA...tu_clave_aqui...
OPENAI_API_KEY=sk-...tu_clave_aqui...

# Configuraciones opcionales
LOG_LEVEL=INFO
CACHE_TTL_SECONDS=3600
```

### Paso 4: Preparar Archivos de Entrada

#### Archivo: `data/user_story.txt`

Crea una descripción clara de la funcionalidad:

```
Como usuario del sistema bancario,
Quiero poder transferir dinero entre mis cuentas,
Para poder gestionar mis finanzas de manera eficiente.
```

#### Archivo: `data/prompt_examples.json`

Ejemplo de estructura (ya viene incluido):

```json
[
  {
    "summary": "Login functionality test",
    "steps": [
      {
        "action": "Navigate to login page",
        "input_data": "URL: /login",
        "expected_result": "Login form displayed"
      }
    ]
  }
]
```

### Paso 5: Verificación

```bash
# Probar configuración
python -c "
from src.config import settings
print(f'✅ Gemini: {\"Configurado\" if settings.gemini_api_key else \"Falta configurar\"}')
print(f'✅ OpenAI: {\"Configurado\" if settings.openai_api_key else \"Falta configurar\"}')
print(f'✅ Archivos: {\"Listos\" if all(Path(f).exists() for f in [\"data/user_story.txt\", \"data/prompt_examples.json\"]) else \"Revisar\"}')
"
```

## 🎮 Uso Interactivo

### Modo Principal: Interactivo (Recomendado)

```bash
python -m src.main
```

### Pantalla de Bienvenida

```
QA Test Generator - Interactive Mode
====================================

Bienvenido al Generador de Casos de Prueba QA
```

### Flujo de User Story

#### Paso 1: Generación Automática
```
=== USER STORY WORKFLOW ===
Generating User Story...
```

El sistema genera una historia de usuario completa en formato SCRUM.

#### Paso 2: Revisión del Contenido
```
Generated User Story:
─────────────────────
ENGLISH VERSION
───────────────
HU-01 - Secure Money Transfer

User Story
──────────
As a banking system user,
I want to securely transfer money between my accounts,
So that I can efficiently manage my finances.

Acceptance Criteria (Gherkin)
─────────────────────────────
1. Given I am logged into the banking system
   When I initiate a transfer between my accounts
   Then the transfer should complete successfully
[... continúa ...]
```

#### Paso 3: Decisión de Aceptación
```
Do you accept this User Story? (y/N): █
```

**Opciones:**
- **`y` + Enter**: Aceptar y continuar
- **`n` + Enter**: Rechazar y modificar

#### Paso 4: Modificación (si rechazas)
```
User Story not accepted. Proceeding to modification...

✅ User Story saved to: output/user_story_for_modification.txt

Please edit the file and save your changes.
Press Enter when you have finished editing output/user_story_for_modification.txt...
```

**Proceso:**
1. El sistema abre tu editor de texto
2. Modifica el archivo según necesites
3. Guarda los cambios
4. Presiona Enter en la terminal
5. El sistema regenera con tus modificaciones
