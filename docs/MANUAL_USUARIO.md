## 🎯 Introducción

### ¿Qué es QA Test Generator?

El **QA Test Generator** es una herramienta profesional de línea de comandos que utiliza inteligencia artificial para automatizar la creación de casos de prueba QA a partir de historias de usuario. El sistema genera documentación completa en formato SCRUM, soportando flujos de trabajo en inglés y español.

### 🎯 Propósito
Transformar el proceso manual de creación de casos de prueba en uno automatizado e inteligente, manteniendo alta calidad y soportando equipos bilingües.

### 💡 ¿Para quién está diseñado?
- **QA Engineers**: Para automatizar la creación de casos de prueba
- **Product Owners**: Para validar historias de usuario
- **Scrum Masters**: Para asegurar calidad en procesos ágiles
- **Desarrolladores**: Para generar casos de prueba durante el desarrollo
- **Cualquier persona de la compañía**: Que necesite crear documentación de testing de manera eficiente

### ⏱️ Información General
- **Tiempo de lectura**: 25 minutos
- **Nivel**: Usuario intermedio
- **Requisitos**: Python 3.9+, claves API de IA
- **Idiomas soportados**: Español e inglés
- **Plataformas**: Windows, Linux, macOS

## ⚙️ Instalación y Configuración

### Paso 1: Requisitos del Sistema

| Componente | Versión Mínima | Recomendado | Notas |
|------------|----------------|-------------|-------|
| **Python** | 3.9.0 | 3.11+ | Versión base |
| **RAM** | 4GB | 8GB+ | Para procesamiento de IA |
| **Almacenamiento** | 500MB | 1GB+ | Para modelos y caché |
| **Internet** | Estable | Alta velocidad | Conexión a APIs de IA |
| **SO** | Windows/Linux/macOS | Windows 11+ | Totalmente compatible |

### Paso 2: Instalación

#### Opción A: Instalación Estándar (Recomendado)
```bash
# 1. Clonar el repositorio
git clone <url-del-repositorio>
cd qa-test-generator

# 2. Crear entorno virtual (opcional pero recomendado)
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # Linux/macOS

# 3. Instalar dependencias
pip install -r requirements.txt

# 4. Verificar instalación
python -c "import src.main; print('✅ Instalación exitosa')"
```

#### Opción B: Usando Poetry
```bash
# Instalar Poetry si no lo tienes
curl -sSL https://install.python-poetry.org | python3 -

# Instalar dependencias
poetry install

# Activar entorno virtual
poetry shell
```

### Paso 3: Configuración de APIs de IA

#### Obtener Claves API

**Para Google Gemini:**
1. Ve a [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Inicia sesión con tu cuenta Google
3. Crea una nueva API key
4. Copia la clave generada

**Para OpenAI:**
1. Ve a [OpenAI Platform](https://platform.openai.com/api-keys)
2. Inicia sesión en tu cuenta
3. Crea una nueva API key
4. Copia la clave (formato: sk-...)

#### Configurar Variables de Entorno

```bash
# Crear archivo .env
cp .env.example .env

# Contenido del archivo .env
GEMINI_API_KEY=AIzaSyA...tu_clave_aqui...
OPENAI_API_KEY=sk-...tu_clave_aqui...

# Configuraciones opcionales
LOG_LEVEL=INFO
CACHE_TTL_SECONDS=3600
GEMINI_REQUESTS_PER_MINUTE=60
OPENAI_REQUESTS_PER_MINUTE=60
```

### Paso 4: Preparar Archivos de Entrada

#### Archivo: `data/user_story.txt`
Crea una descripción clara de la funcionalidad deseada:

```txt
Como usuario del sistema bancario,
Quiero poder transferir dinero entre mis cuentas,
Para poder gestionar mis finanzas de manera eficiente.
```

**Consejos para escribir buenas historias de usuario:**
- Sé específico sobre la funcionalidad
- Incluye el beneficiario (usuario/rol)
- Describe el valor o beneficio
- Mantén un lenguaje claro y conciso

#### Archivo: `data/prompt_examples.json`
Contiene ejemplos de casos de prueba para guiar la generación. Viene pre-configurado, pero puedes modificarlo.

### Paso 5: Verificación de Configuración

```bash
# Verificar configuración completa
python -c "
from src.config import settings
print(f'✅ Gemini: {\"Configurado\" if settings.gemini_api_key else \"Falta configurar\"}')
print(f'✅ OpenAI: {\"Configurado\" if settings.openai_api_key else \"Falta configurar\"}')
print(f'✅ Archivos: {\"Listos\" if all(Path(f).exists() for f in [\"data/user_story.txt\", \"data/prompt_examples.json\"]) else \"Revisar\"}')"
```

## 🎮 Modos de Uso

### Modo Interactivo (Recomendado para Principiantes)

Ideal para usuarios que quieren control total sobre el proceso de generación.

```bash
python -m src.main
```

**Características:**
- Guía paso a paso
- Vista previa de contenido antes de aceptar
- Opciones de modificación
- Retroalimentación en tiempo real

### Modo Programático (Para Automatización)

Para integración en scripts o pipelines de CI/CD.

```python
from src.workflows.workflow_manager import WorkflowManager

# Inicializar administrador
manager = WorkflowManager()

# Ejecutar flujo completo
context = manager.execute_complete_workflow(
    user_story_path="data/user_story.txt",
    examples_path="data/prompt_examples.json",
    interactive=False
)

# Verificar resultados
if context.state.value == "completed":
    print("✅ Éxito!")
    print(f"Historia de usuario: {len(context.final_user_story)} caracteres")
    print(f"Casos de prueba: {len(context.final_test_cases)} caracteres")
```

### Integración en Scripts de Automatización

```python
from src.workflows.workflow_manager import WorkflowManager
import os

def generate_qa_docs(user_story_file, output_dir="output"):
    """Genera documentación QA completa desde un archivo de historia de usuario"""

    # Inicializar manager
    manager = WorkflowManager()

    # Ejecutar workflow completo
    context = manager.execute_complete_workflow(
        user_story_path=user_story_file,
        examples_path="data/prompt_examples.json",
        interactive=False
    )

    # Verificar resultado
    if context.state.value == "completed":
        print("✅ Generación exitosa!")
        print(f"📄 Historia: {len(context.final_user_story)} caracteres")
        print(f"🧪 Casos: {len(context.final_test_cases)} caracteres")

        # Mover archivos a directorio deseado
        os.makedirs(output_dir, exist_ok=True)
        # ... lógica para mover archivos ...

        return True
    else:
        print("❌ Error en generación")
        return False

# Uso
if __name__ == "__main__":
    success = generate_qa_docs("data/user_story.txt", "docs/qa")
    exit(0 if success else 1)
```

## ❗ Solución de Problemas

### Problemas Comunes y Soluciones

#### Error: "API Key not configured"

**Síntomas:**
```
Error: GEMINI_API_KEY environment variable not set
```

**Soluciones:**
1. Verificar que el archivo `.env` existe
2. Confirmar que las claves API están correctamente configuradas
3. Reiniciar la terminal si es necesario
4. Verificar que no hay espacios extra en las variables

#### Error: "Rate limit exceeded"

**Síntomas:**
```
Error: Rate limit exceeded for Gemini API
```

**Soluciones:**
1. Esperar unos minutos antes de reintentar
2. Reducir `REQUESTS_PER_MINUTE` en configuración
3. Cambiar a otro proveedor de IA
4. Implementar backoff exponencial en scripts

#### Error: "File not found"

**Síntomas:**
```
Error: data/user_story.txt not found
```

**Soluciones:**
1. Verificar que el archivo existe en la ruta correcta
2. Crear el archivo con contenido válido
3. Usar rutas absolutas si es necesario
4. Verificar permisos de lectura

#### Error: "Connection timeout"

**Síntomas:**
```
Error: Connection timeout when calling AI API
```

**Soluciones:**
1. Verificar conexión a internet
2. Intentar con el otro proveedor de IA
3. Aumentar timeouts en configuración
4. Reintentar la operación

## 💡 Mejores Prácticas

### Escritura de Historias de Usuario

#### Estructura Recomendada

```txt
Como [tipo de usuario],
Quiero [funcionalidad deseada],
Para [beneficio o valor obtenido].
```

**Ejemplos de buenas historias:**

✅ **Buena:**
```
Como cliente registrado,
Quiero restablecer mi contraseña,
Para poder acceder a mi cuenta si olvido la contraseña.
```

❌ **Mala (muy técnica):**
```
Como usuario,
Quiero que el sistema envíe un token JWT,
Para autenticarme vía API REST.
```

#### Consejos para Historias Efectivas

1. **Enfocarse en el usuario, no en la tecnología**
2. **Incluir el valor o beneficio**
3. **Mantener lenguaje simple y claro**
4. **Evitar detalles técnicos innecesarios**
5. **Ser específico pero no demasiado detallado**

### Generación de Casos de Prueba

#### Cobertura Completa

**Tipos de casos a considerar:**
- **Casos positivos**: Funcionamiento normal
- **Casos negativos**: Manejo de errores
- **Casos límite**: Valores extremos
- **Casos de borde**: Condiciones especiales
- **Casos de seguridad**: Validación de permisos

#### Estructura de Casos de Prueba

**Elementos esenciales:**
1. **ID único**: TC-001, TC-002, etc.
2. **Resumen claro**: Qué se está probando
3. **Precondiciones**: Estado inicial requerido
4. **Pasos detallados**: Secuencia exacta
5. **Resultados esperados**: Qué debe suceder
6. **Prioridad**: Alta, Media, Baja
7. **Tipo**: Positivo, Negativo, Exploratorio

# Estrategia de fallback automático
def generate_with_fallback(content, primary_provider="gemini"):
    providers = ["gemini", "openai"]
    if primary_provider not in providers:
        primary_provider = providers[0]

    for provider in providers:
        try:
            return generate_content(content, provider)
        except Exception as e:
            logger.warning(f"Provider {provider} failed: {e}")

    raise Exception("All providers failed")

## ❓ Preguntas Frecuentes

### Sobre la Instalación

**P: ¿Es compatible con Windows/Linux/macOS?**
R: Sí, completamente compatible con los tres sistemas operativos principales.

**P: ¿Puedo usar solo un proveedor de IA?**
R: Sí, puedes configurar solo Gemini o solo OpenAI según tus necesidades.

**P: ¿Necesito conocimientos de Python para usar la herramienta?**
R: No, el modo interactivo no requiere conocimientos de programación.

### Sobre la Generación de Contenido

**P: ¿Cómo puedo mejorar la calidad de las historias de usuario generadas?**
R: Proporciona descripciones más detalladas y específicas en el archivo de entrada. Incluye contexto del negocio y requisitos específicos.

**P: ¿Los casos de prueba cubren todos los escenarios posibles?**
R: La IA genera casos basados en patrones comunes y mejores prácticas, pero siempre revisa y ajusta según tus necesidades específicas.

**P: ¿Puedo modificar los prompts utilizados por la IA?**
R: Sí, puedes personalizar las plantillas en `src/prompts/` para adaptarlas a tus estándares corporativos.

### Sobre Rendimiento y Costos

**P: ¿Cuánto cuesta usar la herramienta?**
R: Depende del proveedor de IA y la cantidad de contenido generado. Gemini es generalmente más económico que OpenAI.

**P: ¿Cómo puedo reducir los tiempos de generación?**
R: Usa el caché activado, reduce la complejidad de las historias de usuario, y considera usar Gemini para respuestas más rápidas.

**P: ¿Hay límites de uso?**
R: Sí, cada proveedor tiene límites de rate limiting. La herramienta los respeta automáticamente.

### Sobre Personalización

**P: ¿Puedo integrar esto con Jira o Azure DevOps?**
R: Sí, puedes usar el modo programático para integrar con otras herramientas mediante APIs.

**P: ¿Es posible generar casos de prueba en otros formatos además de CSV?**
R: Actualmente soporta CSV, pero puedes extender la funcionalidad en `src/utils/output_handler.py`.

**P: ¿Puedo agregar más proveedores de IA?**
R: Sí, implementa una nueva clase en `src/agents/` siguiendo el patrón del `BaseAgent`.

### Sobre Solución de Problemas

**P: ¿Qué hago si la IA genera contenido incorrecto?**
R: Modifica el archivo de entrada para ser más específico, o usa el modo interactivo para corregir manualmente.

**P: ¿Cómo puedo debug problemas técnicos?**
R: Revisa los logs en `logs/`, aumenta el nivel de logging a DEBUG, y usa el script de diagnóstico.

**P: ¿La herramienta guarda mi información sensible?**
R: No, todas las claves API se mantienen locales y no se transmiten a ningún servidor externo excepto las APIs oficiales de IA.
