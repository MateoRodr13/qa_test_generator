## 🎯 Introduction

The **QA Test Generator** is a command-line tool that uses artificial intelligence to automatically generate QA user stories and test cases in SCRUM format, supporting both English and Spanish.

### 🎯 Level: Intermediate User
### 🛠️ Requirements: Python 3.9+, AI API keys

## 🚀 Quick Start

### Get Ready in 5 Steps

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Configure API keys
cp .env.example .env
# Edit .env with your API keys

# 3. Prepare input files
# Create data/user_story.txt with your user story
# Verify data/prompt_examples.json

# 4. Run
python -m src.main

# 5. Done! Follow on-screen instructions
```

**Expected result**: Files generated in `output/` with user stories and test cases in EN/ES.

## ⚙️ Installation and Configuration

### Step 1: System Requirements

| Component | Minimum Version | Recommended |
|-----------|----------------|-------------|
| **Python** | 3.9.0 | 3.11+ |
| **RAM** | 4GB | 8GB+ |
| **Storage** | 500MB | 1GB+ |
| **Internet** | Stable | High speed |

### Step 2: Installation

#### Option A: Using pip (Recommended)
```bash
# Clone repository
git clone <repository-url>
cd qa-test-generator

# Install dependencies
pip install -r requirements.txt

# Verify installation
python -c "import src.main; print('✅ Installation successful')"
```

#### Option B: Using Poetry
```bash
# Install Poetry if you don't have it
curl -sSL https://install.python-poetry.org | python3 -

# Install dependencies
poetry install

# Activate virtual environment
poetry shell
```

### Step 3: API Keys Configuration

#### Obtaining API Keys

**For Google Gemini:**
1. Go to [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Create a new API key
3. Copy the key

**For OpenAI:**
1. Go to [OpenAI Platform](https://platform.openai.com/api-keys)
2. Create a new API key
3. Copy the key

#### Environment Variables Setup

```bash
# Create .env file
cp .env.example .env

# Edit .env with your keys
GEMINI_API_KEY=AIzaSyA...your_key_here...
OPENAI_API_KEY=sk-...your_key_here...

# Optional configurations
LOG_LEVEL=INFO
CACHE_TTL_SECONDS=3600
```

### Step 4: Input Files Preparation

#### File: `data/user_story.txt`

Create a clear description of the functionality:

```txt
As a banking system user,
I want to securely transfer money between my accounts,
To efficiently manage my finances.
```

#### File: `data/prompt_examples.json`

Example structure (already included):

```json
[
  {
    "summary": "Login functionality test",
    "steps": [
      {
        "action": "Navigate to login page",
        "input_data": "URL: /login",
        "expected_result": "Login form displayed"
      }
    ]
  }
]
```

### Step 5: Verification

```bash
# Test configuration
python -c "
from src.config import settings
print(f'✅ Gemini: {\"Configured\" if settings.gemini_api_key else \"Not configured\"}')
print(f'✅ OpenAI: {\"Configured\" if settings.openai_api_key else \"Not configured\"}')
print(f'✅ Files: {\"Ready\" if all(Path(f).exists() for f in [\"data/user_story.txt\", \"data/prompt_examples.json\"]) else \"Check\"}')"
```

### User Story Workflow

#### Step 1: Automatic Generation
```txt
=== USER STORY WORKFLOW ===
Generating User Story...
```

The system generates a complete user story in SCRUM format.

#### Step 2: Content Review
```txt
Generated User Story:
─────────────────────
ENGLISH VERSION
───────────────
HU-01 - Secure Money Transfer

User Story
──────────
As a banking system user,
I want to securely transfer money between my accounts,
So that I can efficiently manage my finances.

Acceptance Criteria (Gherkin)
─────────────────────────────
1. Given I am logged into the banking system
   When I initiate a transfer between my accounts
   Then the transfer should complete successfully
[... continues ...]
```

#### Step 3: Acceptance Decision
```txt
Do you accept this User Story? (y/N): █
```

**Options:**
- **`y` + Enter**: Accept and continue
- **`n` + Enter**: Reject and modify

#### Step 4: Modification (if rejected)
```txt
User Story not accepted. Proceeding to modification...

✅ User Story saved to: output/user_story_for_modification.txt

Please edit the file and save your changes.
Press Enter when you have finished editing output/user_story_for_modification.txt...
```

**Process:**
1. The system opens your text editor
2. Modify the file as needed
3. Save changes
4. Press Enter in terminal
5. System regenerates with your modifications
